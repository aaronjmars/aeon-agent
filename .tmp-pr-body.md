## What
New `syndicate-article` skill that auto-cross-posts Aeon articles to Dev.to for organic developer discovery. Articles are published with canonical URLs pointing back to the GitHub Pages gallery, preserving SEO attribution.

## Why
Aeon generates high-quality articles daily (repo recaps, deep research, technical explainers) and publishes them to GitHub Pages. But GitHub Pages only reaches users who already know the project. Dev.to has 1M+ active developer readers — a syndication skill multiplies article reach by 10x with zero changes to the existing content pipeline. This was the #2 highest-impact idea from the Apr 14 repo-actions run.

## Changes
- `skills/syndicate-article/SKILL.md`: New skill definition — selects the most recent article, parses title/body/tags from filename conventions, builds canonical URL matching Jekyll permalink pattern, posts via Dev.to API with WebFetch (sandbox-safe) or post-process fallback. Includes 7-day dedup check, graceful skip when DEVTO_API_KEY is absent, and error handling for 401/422 responses.
- `scripts/postprocess-devto.sh`: Sandbox fallback script — processes `.pending-devto/*.json` payloads after Claude finishes, using curl with full env access. Handles success (201), duplicates (422), and errors.
- `dashboard/app/api/secrets/route.ts`: Added `DEVTO_API_KEY` as a builtin secret under new "Distribution" group, with description linking to dev.to/settings/extensions.
- `aeon.yml`: Registered `syndicate-article` skill at 3:30 PM UTC daily (30 min after article-generating skills), using claude-sonnet-4-6 for cost efficiency.
