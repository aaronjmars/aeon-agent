## Self-Improve: fix per-skill model parsing (2026-04-01)
- **Bug found**: push-recap failed on 2026-03-31 with model `claude-sonnet-4-6}` (stray `}`)
- **Root cause**: awk-based per-skill model parser in `.github/workflows/aeon.yml` line 191 scanned forward from the matched skill line until it hit ANY `model:` entry in the file — even from a later skill. For `push-recap` (no model override), it picked up `heartbeat`'s `model: "claude-sonnet-4-6"` and the `tr -d ' }"'` stripping failed to remove the trailing `}`.
- **Fix**: Replaced multi-line awk with `grep -E "^  ${SKILL_NAME}:.*model:"` which only matches on the skill's own line. If no model override, SKILL_MODEL is empty and falls back to global default correctly.
- **Files changed**: `.github/workflows/aeon.yml` (line 191, 2 lines)
